# 文档中心

## 目录结构

| 目录 | 职责 | 负责人 |
|------|------|--------|
| [prd/](prd/) | 产品需求文档：定义"要做什么" | 产品经理 |
| [api/](api/) | API接口契约：OpenAPI规范，前后端共同遵守 | 后端+前端 |
| [design/](design/) | 技术设计文档：定义"怎么做" | 技术负责人 |
| [qa/](qa/) | 质量保证文档：测试用例、验收标准 | QA负责人 |
| [references/](references/) | 业务参考资料：原始材料、政策文件 | 产品经理 |

## 文档流转

```
prd/ (需求定义)
  ↓
api/ (接口契约) ──→ 前后端并行开发
  ↓
design/ (技术方案)
  ↓
qa/ (测试用例)
  ↓
验收上线
```

## 规则

- 所有接口类型必须从 `api/openapi.yaml` 生成，禁止手写类型
- PRD 定稿后才能生成 API Mock
- 技术设计文档必须引用对应 PRD 和 API 定义
- 业务原始材料统一放在 references/ 下，按业务类型分子目录
