# PRD（产品需求文档）

## 职责

定义产品"要做什么"——不涉及技术实现。

## 存放内容

- **指导思想**：产品定位、核心功能规格、设计原则
- **PRD 文档**：背景与目标、用户与场景、功能范围、GWT 场景、验收标准、风险假设
- **mock/**：PRD 定稿后生成的 API Mock（openapi.yaml），由 GWT 场景映射而来

## 边界

| 是 | 不是 |
|----|------|
| 用户场景、功能边界、验收标准 | 技术架构、数据模型、API 实现 |
| 为什么要做、做给谁用 | 怎么做、用什么技术栈 |
| 每个功能的 GWT 正常/异常场景 | 代码实现细节 |

## 负责人

产品经理
