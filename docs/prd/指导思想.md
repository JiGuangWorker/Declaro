# 材料制作平台 — 指导思想

> 版本：v1.0 | 日期：2026-07-07

## 产品一句话

一个手机端工具，把复杂资质材料的制作流程拆成一步步可模仿的小任务，帮业主填得对、拍得准、用得方便。

---

## 产品定位

| 维度 | 说明 |
|------|------|
| 品类 | 材料制作工具，不是业务审批系统 |
| 服务对象 | 需要办理经营许可资质的业主（个体户、小企业主） |
| 核心价值 | 降低认知门槛——让不会办的人也能独立完成材料准备 |
| 端侧 | 手机端优先，填表单、拍照上传、AI 检测全在手机上完成 |
| 业务边界 | 只负责材料制作环节，不涉及盖章、审批、提交等后续流程 |

---

## 业务与平台的关系

具体业务（农药经营许可、食品经营许可等）是**可定制的模板**，平台本身是**通用引擎**。

- 平台提供：步骤引导、表单填报、文件上传、AI 质检、组合导出等通用能力
- 业务模板定义：材料项目清单、每个项目的填报规则、注意事项、参考示例
- 新增一个业务 = 配置一套新模板，不修改平台代码

> 当前首个模板：**农药经营许可**

### 后台材料模板可配置

平台需要配套一个**后台管理端**，用于配置和管理材料模板，这是平台"通用引擎"定位的关键支撑。

**模板配置能力：**

| 配置项 | 说明 |
|--------|------|
| 模板基本信息 | 模板名称（如"农药经营许可"）、所属业务类型、启用/停用状态 |
| 材料项目定义 | 一个模板下包含哪些材料项目，支持增删、排序、拖拽调整顺序 |
| 项目类型 | 每个项目标记为"表单填报"或"文件上传"或"两者都有" |
| 表单字段 | 每个项目下的具体字段（如：姓名、地址、电话），定义字段类型、是否必填、格式校验规则 |
| 注意事项 | 每个项目/字段级别的提示文字，告诉用户怎么填、注意什么 |
| 参考示例 | 每个项目可上传参考图片/示例文字，用户在手机端填报时可对照查看 |
| 质检规则 | 每个上传类项目可配置 AI 检测标准（如：需包含警示牌、需文字清晰可辨） |

**实现方式：Form Schema + Render**

后台将模板配置输出为一份 **Form Schema**（JSON 结构），前端通过统一的 **Render 引擎** 解析 Schema 并动态渲染出表单页面。

```
后台配置模板 → 生成 Form Schema (JSON) → 前端 Render 引擎解析 → 动态渲染表单
```

- 新增一个业务模板 = 后台配置一份 Schema，前端零改动
- 修改模板的字段、校验规则、注意事项 → 更新 Schema 即可，手机端实时生效
- 同一套 Render 引擎适配所有业务模板，保证交互一致性

**设计要点：**

- 一个模板可被多个业主同时使用，互不干扰
- 模板支持版本管理，修改后不影响已生成的存量材料

## 核心功能规格

### 1. 步骤引导
- 每个材料项目为一个独立步骤
- 用户始终知道"当前在哪一步、下一步做什么、还剩多少步"
- 支持顺序推进，也支持跳转到任意步骤修改

### 2. 表单填报
- 每个步骤包含需要填写的信息字段
- 每个字段附带**注意事项**（说明怎么填、填什么格式）
- 每个步骤附带**参考示例**（图片或文字，让用户一眼看懂标准）

### 3. 多材料项目上传与独立管理
- 一个材料模板包含多个材料项目（如：警示牌照片、场地证明、制度文件等）
- 每个项目支持上传多张图片/文件
- 项目之间独立管理，互不影响，可随时增删改

### 4. AI 拍照质检
- 用户拍照上传材料后，后端 AI 自动检测质量
- 检测维度：清晰度、完整性、关键要素是否可见
- 合格 → 标记通过，进入下一步
- 不合格 → 明确提示哪里有问题，引导用户重新拍摄

### 5. 灵活组合导出
- 支持**单选导出**：下载某一个材料项目（图片/PDF）
- 支持**多选导出**：勾选多个项目，合并导出
- 支持**全部打包**：所有材料项目打包为一个 PDF 文件
- 导出后用户自行处理（打印、盖章、提交等），平台不参与后续

---

## 设计原则

1. **示范化**：每个步骤都有"这样就算合格"的可视化参考，用户照做就行
2. **小步快走**：把大任务拆成小步骤，每步只关注一件事，降低心理负担
3. **即时反馈**：拍照马上检测，填错马上提示，不让用户带着问题往下走
4. **用户可控**：材料项目随时可增删改，导出随心组合，平台不替用户做决定
5. **手机原生体验**：所有操作在手机上流畅完成，不依赖 PC 端

---

## 不做什么（本期）

- 不做用户注册/登录体系（MVP 阶段可后置）
- 不做审批流程、不做盖章功能
- 不做材料提交到政府端的能力
- 不做 PC 端适配（手机端优先）
- 不处理具体业务逻辑（那是模板的事，不是平台的事）
